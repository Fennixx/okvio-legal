Okvio Privacy Policy

Okvio is a private-by-default personal-finance app for macOS.

By default, everything stays on your Mac. AI always runs locally. The optional iCloud sync is end-to-end encrypted, so nobody, not even Apple, can read your data.

Where data lives

  • A local SQLite database (GRDB) in the app's sandbox container.
  • The unlock PIN is stored only as a PBKDF2-HMAC-SHA256 hash plus salt in the device Keychain (…WhenUnlockedThisDeviceOnly). It never leaves the device.
  • Bank statements you upload are parsed on device (text layer or Vision OCR). The file is read via a user-selected, security-scoped grant and is not copied off device.

AI and model providers: on-device only

The AI layer (narration and categorization) is on-device by design. There is no cloud AI provider. All numbers come from the deterministic core and are verified by a numeric firewall, so the model only phrases pre-computed facts.

  • Apple Intelligence (default): Apple's on-device Foundation Models. If the chosen language isn't supported, a deterministic Czech or English template floor is used.
  • Ollama (local): a model server running on this Mac. The app only ever connects to loopback (localhost). The endpoint policy rejects everything else, so transaction text cannot leave the machine.

The app declares the network.client entitlement only to reach the local Ollama endpoint. With the on-device default it makes no network calls.

iCloud sync (opt-in)

Multi-device sync is strictly opt-in (Settings, then iCloud Sync, off by default). When enabled:

  • Every record is encrypted on this Mac with AES-GCM before upload. The 256-bit key lives only in iCloud Keychain (itself end-to-end encrypted) and is never sent to Okvio or anyone else.
  • CloudKit stores opaque blobs in your private database, readable by your devices and by no one else, regardless of your Advanced Data Protection setting. Each payload is bound to its record identity, so even a server-side swap of blobs fails to decrypt.
  • Metadata visible to Apple: record counts, record identifiers (table name plus random UUID), and sync timing. Never contents, amounts, merchants, or notes.
  • Turning sync off keeps all local data and stops all network traffic. "Reset iCloud data" deletes the cloud copy and re-uploads from this Mac (the recovery path after an iCloud Keychain reset).

Until you enable it, nothing syncs and everything stays local.

Household sharing (opt-in)

You can share selected accounts with a partner as a separate space (your private data never mixes in). Sharing reuses the same end-to-end-encrypted channel:

  • The shared space gets its own household key, generated on this Mac. It travels to your partner inside the invite link (over your own channel, e.g. iMessage), never through CloudKit. Apple stores only opaque blobs for shared data too.
  • The share is closed: only the partner's invited Apple ID (by email or phone) can accept, and decryption needs the in-link key, so a leaked link alone is useless.
  • Both participants edit shared data, and each change records who made it.
  • The owner can remove a participant at any time: the key is rotated and the data re-uploaded, so old links and cached copies stop working. A removed member keeps their local copy but no longer syncs.
  • A member can leave, and the owner can stop sharing. Private accounts always stay private.

Permissions

  • Face ID and device authentication (optional): to unlock the app.
  • Notifications (optional, off by default): local budget-threshold alerts and planned-payment reminders only.

Export compliance

Okvio uses only standard, Apple-provided cryptography (Keychain, CryptoKit AES-GCM for the opt-in end-to-end-encrypted sync, and PBKDF2 hashing). It implements no proprietary or non-exempt encryption.
